Gold Layer · KPI Health + Peer Benchmark

databricks/notebooks/gold/01_build_kpi_health.py Pipeline
Computes 6-month trend slope, 12-month volatility, persistence (consecutive declining months), deviation from seasonal baseline → assigns health_status label
Reads silver.internal_metrics. Every (asset, metric) combo gets: 'good', 'review', or 'stable'. Feeds priority scoring.
+
Called From
silver.internal_metrics (all 103 months)
Calls Next
analytics/02_compute_priority_inputs.pygold/04_build_priority_board.py
Key Formulas
Trend: REGR_SLOPE() on last 6 months · Volatility: STDDEV/AVG over 12m window · Persistence: custom consecutive-decline counter · Deviation: (value - 12m_avg) / 12m_avg
Health Rule
slope > 0.05 AND vol < 0.2 → 'good' · slope < -0.05 OR persistence ≥ 3 OR dev > 20% → 'review' · else 'stable'
Output Table
gold.kpi_health — 2,184 rows (103 months × 4 assets × ~53 metrics)
databricks/notebooks/gold/02_build_peer_benchmark.py Pipeline
Joins Real Madrid vs 5 peer clubs on 8 shared metrics → computes peer median, leader value, RM rank (1-6), gap to median (polarity-adjusted)
Critical: polarity handling via metric_dictionary.json. bounce_rate is the only -1 polarity metric (lower is better → logic inverts).
+
Called From

Calls Next
analytics/02_compute_priority_inputs.py
8 Metrics
website: unique_visitors, visits, bounce_rate, recurrence · ecommerce: unique_visitors, visits, conversion_rate, net_sales
Key Formula
gap = (rm_value - peer_median) × polarity. Polarity -1 for bounce_rate: positive gap still means worse than peers.
Output Table
gold.peer_benchmark — 824 rows (8 metrics × 103 months)
Signal Detection + Priority Scoring
4
Analytics · Signal Detection
Lagged Pearson correlation · Business prior gate · 60% threshold · ~20 validated signals from ~2,700 possible pairs
databricks/notebooks/analytics/01_validate_signals.py Analytics
Tests every possible metric pair at 1/2/3 month lags using Pearson correlation → filters by |corr| > 0.6 → applies business prior gate → writes validated signals
~2,700 pairs tested, ~10-20 survive. Example: website_unique_visitors → ecommerce_net_sales (lag 2m, corr 0.72). Business prior: "sales→traffic" is reverse causality → rejected.
+
Called From

Calls Next
